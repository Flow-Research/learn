# WorkStream

The task and value pipeline. WorkStream is where useful work enters Flow, gets distributed, verified, and rewarded.

## What it does

WorkStream takes tasks from Economic Value Sources and moves them through five stages:

1. **Decomposition** — breaking work into smaller chunks
2. **Distribution** — routing tasks to humans and Personal Operators
3. **Verification** — checking output quality
4. **Aggregation** — combining accepted work
5. **Attribution** — handling rewards and reputation

## How it fits

WorkStream gives the agent and human valuable work. It makes contribution visible, verifiable, and rewardable.
